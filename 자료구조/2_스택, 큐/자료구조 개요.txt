# 자료구조&알고리즘(Data Structure)
자료구조는 컴퓨터 과학에서 효율적인 접근 및 수정을 가능케 하는 자료의 조직, 관리, 저장을 의미한다.
자료구조는 데이터 값의 모임, 데이터 간의 관계, 그리고 데이터에 적용할 수 있는 함수나 명령을 의미한다.
파이썬에서 제공하는 기본 자료구조인 리스트, 튜플, 딕셔너리는 사용하기 편리하며 데이터를 다루는 데도 효과적이다.
@ 위키백과 자료구조&알고리즘 "다양한~"

# 선형 리스트 - 2_스택, 큐
데이터를 임시 저장할 때 사용하는 자료구조로 스택과 큐가 있다.
데이터 삽입과 삭제의 시간복잡도가 O(1)

# 스택(Stack) &last 어원 네이버 두가지
스택은 데이터를 쌓아 올린 형태의 자료구조이다. 꼬치
LIFO(Last In First Out, 후입선출): 가장 마지막(최근)에 넣은 데이터를 우선적으로 꺼낸다.
push: 데이터를 넣는 작업
pop: 데이터를 꺼내는 작업
top: 데이터 맨 윗부분
bottom: 데이터 맨 아랫부분
ex) 실행취소(ctrl+z), 웹페이지 뒤로가기(방문기록)

# 큐(Queue) &큐 사전검색
큐는 데이터를 줄 세워놓은 형태의 자료구조이다. 먼저 줄 선 사람이 입장한다.
FIFO(First In First Out, 선입선출): 가장 먼저 넣은 데이터를 우선적으로 꺼낸다. &편의점 선입선출
enqueue: 데이터를 넣는 작업
dequeue: 데이터를 꺼내는 작업
front: 데이터를 꺼내는 부분
rear: 데이터를 넣는 부분
ex) 대기열 관리(수강신청), 프린터 출력, 입출력 스트림 버퍼링(버퍼에 쌓아두고 앞에서 부터 차례대로 접근)

# 덱(Deque = double-ended queue)
앞과 뒤에서 모두 삽입과 삭제가 가능한 큐 자료구조이다.
스택과 큐를 합쳐 놓은 자료구조이기도 하다.

addFront: 맨앞 데이터 꺼내기
addRear: 맨뒤 데이터 꺼내기
deleteFront: 맨앞 데이터 삭제
deleteRear: 맨뒤 데이터 삭제
